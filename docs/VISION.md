```
K = 100 × cuberoot(D × O × L)

D = Diamond Hands (conviction)
O = Organic Growth (distribution)
L = Longevity (survival)

→ "Ce token est-il legitime?"
```

### Q-Score (Judgment Quality) - CYNIC calcule

```
Q = 100 × fourthroot(phi × V × C × B)

phi = PHI axiom score
V   = VERIFY axiom score
C   = CULTURE axiom score
B   = BURN axiom score

→ "Ce judgment est-il fiable?"
```

### E-Score (User Reputation) - CYNIC calcule

```
E = sum(dimension × phi_weight)

HOLD   (1.0)   Token holdings
BURN   (phi)   Cumulative burns
USE    (1.0)   API usage
BUILD  (phi²)  Code contributions
RUN    (phi²)  Node operation
REFER  (phi)   Active referrals
TIME   (phi^-1) Days active

→ "Cet utilisateur est-il de confiance?"
```

### Final Score

```
Final = min(K-Score, Q-Score)

→ "Le facteur limitant gagne"
```
