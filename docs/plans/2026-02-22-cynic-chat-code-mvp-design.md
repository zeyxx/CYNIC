## Executive Summary

Build a **Chat/Code unified interface** in the webapp where:
- User talks to CYNIC directly
- CYNIC generates code
- User gives feedback
- Learning happens (with human gates)
- Symbiosis emerges over time

**Scope:** 3 phases, delivered incrementally
- **Phase 1:** Chat UI + Code display (MVP works)
- **Phase 2:** Learning gate (human validation)
- **Phase 3:** Measurement dashboard (symbiosis visible)

**Key insight:** Chat logic EXISTS (CLI). We're porting to web + adding human validation layers.

## Data Flow

### User Interaction Sequence

```
1. User opens webapp
   → sees Chat panel (left) + empty Code panel (right)

2. User types: "Create an API endpoint for user auth"
   → sends POST /api/chat/message {text, session_id}

3. CYNIC responds with: {code: "...", status: "generated", judgment_id: "..."}
   → frontend displays code in right panel

4. User clicks: "Good" / "Needs Fix" / "Show Diff"
   → sends POST /api/learn {event, feedback}

5. Event queued to database (awaiting Senior Dev review)
   → learning agent stores without applying yet

6. Senior Dev reviews (Phase 2)
   → approves/rejects patterns

7. DevOps deploys approved patterns (Phase 2)
   → CYNIC learns + improves
```

  senior_dev_status?: "pending" | "approved" | "rejected";  // Phase 2

## Risk Analysis

**High Risks:**
1. **Integration gaps** — Chat CLI → Webapp port may have issues
   - Mitigation: Empirical testing as we build
2. **Performance** — Chat may be slow with long histories
   - Mitigation: Pagination, compression
3. **Learning quality** — Bad patterns learned early
   - Mitigation: Senior Dev gate (Phase 2)

**Medium Risks:**
1. **Multi-user coordination** (Phase 2+)
   - Mitigation: Single-user MVP, scale later
2. **Measurement accuracy** — Symbiosis metrics hard to define
   - Mitigation: Iterate with user feedback

**Low Risks:**
1. Database schema changes
2. API versioning
