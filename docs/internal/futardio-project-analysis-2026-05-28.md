# Analyse du Projet Futardio — 2026-05-28

**Auteurs :** T. + Claude
**Statut :** Document interne, analyse stratégique

---

## 1. Qu'est-ce que Futardio ?

Convention/événement crypto/blockchain. Opportunité de présenter CYNIC et B&C à un public de builders, investisseurs et communautés crypto francophones.

### Public cible

- Builders Solana (dev teams, protocoles)
- Investisseurs early-stage / business angels crypto
- Communautés crypto francophones
- Institutionnels curieux (banques, fonds)

---

## 2. Analyse SWOT

### Forces

| Force | Détail |
|-------|--------|
| **Produit fonctionnel** | CYNIC tourne en production, 20K+ verdicts, 340+ tests Rust |
| **Code réel** | 82K lignes, 3 langages, 1873 tests — pas du vaporware |
| **Héritage** | HolDex (766 commits), communauté $ASDFASDFA active |
| **Souveraineté** | Infrastructure propre, zéro cloud |
| **Équipe technique** | 3 devs complémentaires (Rust + TS + JS) |
| **Différenciateur** | Aucun concurrent sur le jugement multi-modèle phi-bounded |

### Faiblesses

- Pas d'utilisateurs externes encore
- SAS pas encore créée
- Modèle économique exploratoire
- Équipe jeune, crédibilité à construire

### Opportunités

- Timing : explosion des scam tokens = demande de filtrage
- Niche vacante : jugement AI souverain pour crypto
- BPI : subventions innovation compatibles
- Narrative "agents IA" en pleine explosion

### Risques

- Demo fail (internet instable)
- Public non technique
- Engagement prématuré sur des timelines
- Dilution du message (trop de projets à présenter)

---

## 3. Stratégie

### Deux angles complémentaires

1. **CYNIC (technique)** : pour builders — architecture Rust, multi-modèle, phi-bounded
2. **B&C (business)** : pour investisseurs — SAS, modèle éco, marché adressable

### Messages clés

1. "50K tokens/jour, 98% rugs. On est le filtre qui manque."
2. "3 modèles IA indépendants, zéro cloud, confiance plafonnée à 61.8%."
3. "82K lignes de code, pas un pitch deck — un produit."

### Préparation

- Pitch unifié v6
- Screenshots de fallback
- Demo testée sur 4G
- Script d'antériorité
- ToS draft
- Site web minimal

---

## 4. MetaDAO / Futardio Risk

1. Ne pas promettre de timelines sur features non construites
2. Présenter sous angle unifié (pas deux projets séparés)
3. Architecture = moat par l'exécution, pas par le secret
4. Honnêteté sur les métriques d'usage (pas encore d'utilisateurs externes)
