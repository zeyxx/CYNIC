# CYNIC Kernel Modular Refactor ‚Äî Design Spec

## Goal

Reorganize `cynic-kernel/src/` from 23 flat files into a hexagonal module structure with functional naming (industry standard). Fix 5 architectural violations before reorganization so the new structure encodes clean patterns. Reduce max LOC per file from 896 to ~350.

## Architecture

Hexagonal architecture with functional naming (not academic `adapters/driving/driven/`). Domain layer is sacred ‚Äî zero imports from adapters. Ports defined in domain, adapters implement them. Industrial precedent: 90% of Rust hexagonal projects use functional naming (`api/`, `storage/`, `backends/`).

## Constraints

- All changes must pass `cargo build -p cynic-kernel --release`, `cargo test`, `cargo clippy -- -D warnings`
- Reorganization in a single atomic commit (all imports update together)
- `judge.rs` stays intact (638 LOC, 60% tests, single cohesive pipeline)
- Zero behavioral changes ‚Äî this is a pure structural refactor
- Public repo ‚Äî no secrets, no real IPs
- `cynic_v2` protobuf module stays in `lib.rs` (generated by `tonic::include_proto!`, used by grpc/ and main.rs)

## Violations Inventory

### V1 ‚Äî Cross-adapter coupling: mcp.rs imports DogUsageTracker from rest.rs

**File:** `mcp.rs:28` ‚Äî `use crate::rest::DogUsageTracker;`

**Problem:** Two driving adapters (REST and MCP) coupled through a shared type defined in one of them.

**Fix:** Move `DogUsageTracker` to `domain/usage.rs`. Both REST and MCP import from domain.

### V2+V3 ‚Äî Port bypass: REST and MCP hold Arc\<SurrealHttpStorage\> directly

**Files:**
- `rest.rs` ‚Äî `AppState.raw_db: Option<Arc<SurrealHttpStorage>>`
- `mcp.rs` ‚Äî `CynicMcp.raw_db: Option<Arc<SurrealHttpStorage>>`

**Problem:** `StoragePort` covers verdicts+crystals but NOT coordination (`work_claim`, `agent_session`) or audit (`mcp_audit`). Both adapters bypass the port to run raw SurrealQL.

**Fix:** Create `CoordPort` trait in `domain/coord.rs`:

```rust
#[async_trait]
pub trait CoordPort: Send + Sync {
    async fn register_agent(&self, agent_id: &str, agent_type: &str, intent: &str) -> Result<(), CoordError>;
    async fn claim(&self, agent_id: &str, target: &str, claim_type: &str) -> Result<ClaimResult, CoordError>;
    async fn release(&self, agent_id: &str, target: Option<&str>) -> Result<String, CoordError>;
    async fn who(&self, agent_id_filter: Option<&str>) -> Result<CoordSnapshot, CoordError>;
    async fn expire_stale(&self, ttl_secs: u64) -> Result<(), CoordError>;
    async fn store_audit(&self, tool: &str, agent_id: &str, details: &serde_json::Value) -> Result<(), CoordError>;
    async fn query_audit(&self, tool_filter: Option<&str>, agent_filter: Option<&str>, limit: u32) -> Result<Vec<serde_json::Value>, CoordError>;
}

pub enum ClaimResult {
    Claimed,
    Conflict(Vec<ConflictInfo>),
}

pub struct ConflictInfo {
    pub agent_id: String,
    pub claimed_at: String,
}

pub struct CoordSnapshot {
    pub agents: Vec<serde_json::Value>,
    pub claims: Vec<serde_json::Value>,
}

pub enum CoordError {
    StorageFailed(String),
    InvalidInput(String),
}
```

`SurrealHttpStorage` implements `CoordPort`. REST and MCP receive `Arc<dyn CoordPort>` ‚Äî `raw_db` field eliminated entirely.

### V4 ‚Äî Missing trait: hal.rs uses BackendRouter concretely

**File:** `hal.rs:9` ‚Äî `use crate::router::BackendRouter;`

**Problem:** `MuscleService` depends on concrete `BackendRouter` instead of a trait.

**Fix:** Extract `InferenceRouter` trait in `domain/inference.rs`:

```rust
#[async_trait]
pub trait InferenceRouter: Send + Sync {
    async fn route(&self, req: InferenceRequest) -> Result<InferenceResponse, BackendError>;
    async fn fan_out(&self, req: InferenceRequest, n: u32) -> Result<InferenceResponse, BackendError>;
}
```

`BackendRouter` implements `InferenceRouter`. `MuscleService` uses `Arc<dyn InferenceRouter>`.

### V5 ‚Äî Layer pollution: VascularService defined inline in main.rs

**File:** `main.rs:18-52` ‚Äî `VascularService` struct + impl defined in main.rs.

**Fix:** Move to `grpc/vascular.rs`.

## Execution Order

### Phase 1 ‚Äî Fix violations (3 sequential tasks)

Tasks 1.1 and 1.3 both modify `rest.rs` and `mcp.rs` ‚Äî they must be sequenced, not parallelized. Task 1.2 is independent and can run in parallel with either.

**Execution order:** 1.2 (independent) ‚Üí 1.1 ‚Üí 1.3 (both touch rest.rs/mcp.rs)

**Task 1.1 ‚Äî Fix V1: Move DogUsageTracker to domain**
- Extract `DogUsageTracker` AND its helper struct `DogUsage` from `rest.rs` into standalone file
- Update imports in `rest.rs` and `mcp.rs`
- Commit: `refactor(domain): extract DogUsageTracker from rest adapter`

**Task 1.2 ‚Äî Fix V4: Create InferenceRouter trait** (can run in parallel with 1.1)
- Extract `InferenceRouter` trait (route + fan_out) from `BackendRouter`
- `BackendRouter` implements `InferenceRouter`
- `hal.rs` uses `Arc<dyn InferenceRouter>` instead of `Arc<BackendRouter>`
- `main.rs` creates `Arc<BackendRouter> as Arc<dyn InferenceRouter>`
- Commit: `refactor(domain): InferenceRouter trait ‚Äî decouple hal from BackendRouter`

**Task 1.3 ‚Äî Fix V2+V3: Create CoordPort, eliminate raw_db**
- Create `CoordPort` trait with coord + audit operations
- Implement `CoordPort` for `SurrealHttpStorage`
- Create `NullCoord` for degraded mode (like `NullStorage`)
- Replace `raw_db: Option<Arc<SurrealHttpStorage>>` with `coord: Arc<dyn CoordPort>` in both `AppState` and `CynicMcp`
- Move all raw SurrealQL from REST/MCP into the `CoordPort` implementation
- Update `main.rs` to inject `Arc<dyn CoordPort>`
- Commit: `refactor(domain): CoordPort ‚Äî eliminate raw SurrealHttpStorage bypass`

### Phase 2 ‚Äî Module reorganization (1 atomic commit)

All file moves + import updates in a single commit. No behavioral changes.

**Task 2.1 ‚Äî Create directory structure**
- `mkdir -p src/{domain,dogs,backends,storage,api/rest,api/mcp,grpc,infra,probe}`

**Task 2.2 ‚Äî Move files to target locations**
- domain/: dog.rs, ccm.rs, storage_port.rs‚Üístorage.rs, coord.rs, chat_port.rs‚Üíchat.rs, usage.rs, temporal.rs, inference types from backend.rs
- dogs/: inference_dog.rs‚Üíinference.rs, deterministic_dog.rs‚Üídeterministic.rs
- backends/: backend_openai.rs‚Üíopenai.rs, backend_llamacpp.rs‚Üíllamacpp.rs, router.rs
- storage/: storage_http.rs split into mod.rs + surreal.rs
- grpc/: hal.rs, pulse.rs, vascular.rs (from main.rs)
- infra/: circuit_breaker.rs, config.rs, supervisor.rs

**Task 2.3 ‚Äî Split giants**
- rest.rs ‚Üí api/rest/{mod,types,middleware,judge,data,health}.rs
- mcp.rs ‚Üí api/mcp/{mod,judgment,data,coord}.rs
- probe.rs ‚Üí probe/{mod,types,hardware,environment,llm}.rs

**Task 2.4 ‚Äî Update lib.rs module declarations**

New `lib.rs` structure (replaces 20 flat `pub mod` lines):


This allows most files to use `use crate::domain::*` instead of deep paths.

**Task 2.5 ‚Äî Update all imports in every file**
- Every `use crate::dog::` ‚Üí `use crate::domain::dog::`  (or `use crate::domain::*`)
- Every `use crate::storage_port::` ‚Üí `use crate::domain::storage::`
- main.rs: update all imports, remove VascularService inline
- grpc/ files: import proto types via `use crate::cynic_v2::*`

**Task 2.6 ‚Äî Visibility for storage/ split**
- `SurrealHttpStorage` struct fields (`client`, `url`, `ns`, `db`, `auth`) must be `pub(crate)` or `pub(super)` so `surreal.rs` can access them from `mod.rs`
- Helper functions (`sanitize_id`, `escape_surreal`, `safe_limit`) must be `pub(super)` for `surreal.rs`

**Task 2.7 ‚Äî Verify**
- `cargo build -p cynic-kernel --release`
- `cargo test -p cynic-kernel --release`
- `cargo clippy -p cynic-kernel --release -- -D warnings`
- Commit: `refactor(kernel): hexagonal module structure ‚Äî 23 flat files ‚Üí 9 modules`

## Decisions Log

| Decision | Rationale |
|---|---|
| Functional naming (api/, storage/) not academic (adapters/driving/) | 90% of industrial Rust hexagonal projects use functional naming. Research: openfang, blueprint-hexagonal, LogRocket standard. |
| judge.rs stays intact at 638 LOC | 60% tests (383 LOC). Logic is 255 LOC. Pipeline is cohesive. Splitting creates false modularity. |
| CoordPort as separate trait (not extending StoragePort) | StoragePort = verdicts+crystals (data plane). CoordPort = agent sessions+claims+audit (control plane). Different concerns, different evolution rates. |
| dogs/ as separate module (not inside domain/) | InferenceDog consumes ChatPort via dependency injection. While it contains domain logic, the DI pattern makes it a "use case" layer between pure domain and adapters. |
| probe/ stays as top-level module | Zero CYNIC imports. Self-contained hardware discovery. Not domain, not adapter ‚Äî orthogonal concern. |
| gRPC stubs kept (not deleted) | Infrastructure exists and compiles. Removing it saves nothing and loses optionality. Documented as stubs. |
| BackendStatus stays in domain/inference.rs | Used by both ChatPort.health() and InferencePort.health(). Single definition, two consumers. |
| MockBackend stays in domain/inference.rs | Used by router.rs tests and integration tests. Placing in domain makes it available everywhere without circular deps. |

## Risks

| Risk | Mitigation |
|---|---|
| `probe.rs` split may require exposing internal helpers as `pub(super)` | All functions in probe.rs are module-private. After split, functions in `hardware.rs` called by `mod.rs` need `pub(super)`. This is expected and low-risk. |
| `storage/` split requires visibility changes on `SurrealHttpStorage` fields | Fields become `pub(super)` ‚Äî only visible within the storage module. No public API change. Documented in Task 2.6. |
| `rest.rs` split: private helpers and shared state between handlers | `AppState` stays in `mod.rs`, handlers access it via `axum::extract::State`. Each handler file imports `use super::*` from `mod.rs`. Standard axum pattern. |
| Phase 2 atomic commit touches every file ‚Äî large diff, hard to review | Split review: verify `cargo test` passes (behavioral equivalence), then spot-check import paths. No logic changes = mechanical review. |
| `cynic_v2` proto module import paths | Stays in `lib.rs` exactly as-is. grpc/ files use `use crate::cynic_v2::*`. No change needed. |
