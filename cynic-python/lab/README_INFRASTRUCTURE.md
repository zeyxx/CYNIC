# Python Lab Infrastructure — Data-Centric Multi-Domain Development

## Overview

This infrastructure enables fast iteration on multiple domains in parallel with proper versioning, measurement, and rollback.

**Problem we solved:** Heuristics hardcoded in Python, no version history, changes measured manually or not at all, can't run A/B tests.

**Solution:** Versioned artifacts (JSON) + measurement framework (before/after) + K15 consumer (observation scoring).

## Quick Start

### 1. Establish Baseline for Any Domain

```bash
cd cynic-python/lab
python measure_domain_quality.py \
  --domain twitter \
  --dataset ~/.cynic/organs/hermes/x/dataset.jsonl \
  --limit 100 \
  --baseline

# Output: measurements/twitter_2026-04-30T14-30-00.json
# Shows: confusion matrix, sensitivity, specificity, Pearson r
```

### 2. Change Heuristic

```bash
# Edit the Python code (twitter_dog.py, twitter_signal_extractor.py, etc.)
# Bump version in versions/MANIFEST.yaml
# Document the change in artifacts/twitter_gates_v1.X.json
```

### 3. Measure After Change

```bash
python measure_domain_quality.py \
  --domain twitter \
  --dataset ~/.cynic/organs/hermes/x/dataset.jsonl \
  --limit 100 \
  --after \
  --compare-to measurements/twitter_2026-04-30T14-30-00.json

# Output shows deltas:
#   Sensitivity delta: +0.045
#   Specificity delta: -0.012
#   Pearson r delta: +0.087
```

### 4. Commit or Revert

- Accept if: sensitivity ↑ and specificity ↑ (or <5% drop in one)
- Revert if: sensitivity drops >5% or specificity drops >5%

## Files and Their Purposes

### `versions/MANIFEST.yaml`
**SSOT for all versioned Dogs, thresholds, baselines.**

Structure:
```yaml
domains:
  twitter:
    current_version: "v1.0"
    artifacts:
      v1.0:
        date: "2026-04-30"
        commit_hash: "eda3153"
        metrics:
          sensitivity: 0.75
          specificity: 0.88
          pearson_r: 0.82
        description: "Initial TwitterDog validation"
```

When you bump a version, update this file with:
- New version number
- Date and git commit hash
- Before/after metrics
- Description of the change

### `artifacts/twitter_gates_v1.0.json`
**Versioned heuristic rules (read by kernel + consumer).**

Contains:
- Signal extraction keywords
- Axiom weights
- Verdict thresholds
- Measurement baseline (confusion matrix, sensitivity/specificity)
- High-signal patterns

When you change a heuristic, create a new artifact version and update the version number.

### `measure_domain_quality.py`
**Measurement framework for before/after benchmarking.**

Usage:
```bash
# Baseline
python measure_domain_quality.py --domain twitter --baseline

# After change, compare to baseline
python measure_domain_quality.py --domain twitter --after --compare-to baseline.json
```

Returns JSON with:
- Confusion matrix (TP/FP/TN/FN)
- Sensitivity (recall on BARK)
- Specificity (rejection of non-BARK)
- Pearson r (correlation with ground truth)
- Latency breakdown
- Per-axiom variance

### `measurements/`
**Before/after benchmark results.**

Automatically generated by `measure_domain_quality.py`. Save filenames for comparison in step 3.

### `consumers/k15_observation_consumer.py`
**Closes the K15 producer-consumer loop.**

Runs as systemd service (every 5 minutes):
1. Polls kernel `/observations`
2. Scores with TwitterDog heuristics
3. Filters high-signal (≥3 or known patterns)
4. Dispatches to `/agent-tasks` for Hermes to execute

Wired via: `infra/systemd/hermes-k15-consumer.service`

### `config/domains.yaml`
**Domain definitions and target sizes (D1-D6).**

Used by lab.py to categorize tweets into domains. Versioned (currently v1.1).

### `lab.py`
**Full analysis pipeline.**

Runs against dataset to produce comprehensive briefing:
- Domain distribution
- Signal distribution (high/low)
- False negatives analysis
- Coverage vs ground truth

## Multi-Domain Development Workflow

### Scenario: Working on Twitter + Token domains in parallel

```bash
# Branch 1: Twitter calibration
git checkout -b feat/twitter-calibration-2026-04-30
cd cynic-python/lab
python measure_domain_quality.py --domain twitter --baseline
# ... edit twitter_dog.py ...
python measure_domain_quality.py --domain twitter --after --compare-to [baseline]
git add artifacts/twitter_gates_v1.1.json versions/MANIFEST.yaml
git commit -m "feat(twitter-dog): increase LP-discussion weight → sensitivity +4.5%"

# Branch 2: Token feedback loop (parallel)
git checkout -b feat/token-feedback-loop-2026-04-30
python measure_domain_quality.py --domain token --baseline
# ... wire conviction-only feedback ...
python measure_domain_quality.py --domain token --after --compare-to [baseline]
git add artifacts/token_gates_v1.4.json versions/MANIFEST.yaml
git commit -m "feat(token): add conviction feedback loop → accuracy 98%"
```

Domains don't block each other. Coordinate via MANIFEST.yaml when pushing (git handles conflicts).

## K15 Consumer Deployment

### Install systemd service

```bash
sudo cp infra/systemd/hermes-k15-consumer.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable hermes-k15-consumer.service
sudo systemctl start hermes-k15-consumer.service

# Monitor
journalctl -u hermes-k15-consumer.service -f
```

### Verify it's working

```bash
# Check if observations are being consumed
curl -H "Authorization: Bearer $CYNIC_API_KEY" \
  http://localhost:3030/health | jq '.observations_consumed'

# Should increment over time
```

### Falsification test

```bash
# 1. Check how many observations exist
curl -H "Authorization: Bearer $CYNIC_API_KEY" \
  http://localhost:3030/observations?limit=100 | wc -l

# 2. Let consumer run for 10 minutes
sleep 600

# 3. Verify tasks were created
curl -H "Authorization: Bearer $CYNIC_API_KEY" \
  http://localhost:3030/agent-tasks?kind=hermes | jq '.length'

# Should show 8-10 tasks (assuming 14 high-signal obs)
```

## Measurement Best Practices

1. **Always baseline first.** Never change without a before snapshot.
2. **One variable at a time.** Change one signal/weight, measure, decide.
3. **Use real data.** Test on `dataset.jsonl` (4,146 tweets), not synthetic data.
4. **Accept high bars:**
   - Sensitivity ≥ 0.75 (catch most BARK)
   - Specificity ≥ 0.85 (low false alarm rate)
   - Pearson r ≥ 0.80 (good correlation with ground truth)
5. **Revert aggressively.** If unsure, revert and discuss before next attempt.
6. **Track commits.** Update MANIFEST.yaml with commit_hash for every version.

## Troubleshooting

### Consumer not starting
```bash
# Check logs
journalctl -u hermes-k15-consumer.service -n 50

# Test consumer directly
python cynic-python/consumers/k15_observation_consumer.py \
  --kernel-url http://localhost:3030 \
  --max-iter 1
```

### Measurement fails on dataset
```bash
# Check dataset path exists
ls -la ~/.cynic/organs/hermes/x/dataset.jsonl

# Check format (should be JSONL with 'verdict' field)
head -1 ~/.cynic/organs/hermes/x/dataset.jsonl | python -m json.tool
```

### Heuristic change made things worse
```bash
# Revert the change
git checkout HEAD~ cynic-python/heuristics/twitter_dog.py

# Re-measure
python measure_domain_quality.py --domain twitter --after --compare-to [baseline]

# If reverted metrics match baseline, commit the revert
git commit -m "revert(twitter-dog): sensitivity dropped, reverted weight change"
```
