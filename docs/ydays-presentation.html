Ce qu'on a appris

  • Commencer par la curiosité — un tracker de burns, pas un business plan
  • Accepter que la plupart meurent — 10+ repos en décembre, la plupart abandonnés
  • Changer d'outil quand le problème l'exige — JS → Rust
  • La souveraineté a un coût — maintenir son hardware > payer un cloud

Ce qu'on ferait différemment

  • Moins de repos, plus de profondeur plus tôt
  • Tester sur des vrais utilisateurs avant l'infrastructure
  • Écrire les tests avant le code
