# CYNIC Hermes Architecture — Agentic OS & Sovereign Law

This document defines the reference architecture for the Talaria ecosystem (CYNIC, B&C, etc.), positioning the **NousResearch Hermes-agent** as the central Agentic Operating System.

## 1. The Motor Cortex: Hermes Agentic OS
**Implementation**: `hermes_cli` (Python)
**Reference Status**: PRIMARY AGENTIC OS

- **Nature**: Unlike static bots, Hermes is a recursive system of thought. It functions as an **Agentic OS** that owns:
    - **SOUL.md**: The constitutional mind. It defines the base personality and behavioral boundaries.
    - **Kanban System**: Autonomous task decomposition. Hermes does not just "execute"; it plans, tracks, and re-evaluates via an internal state machine.
    - **Tool Orchestration**: Native Playwright integration (browser), code execution sandboxes, and skill-hub management.
- **Law**: The framework defines the *mechanics* of intelligence. It is the "Brain & Hands".

## 2. The Prefrontal System: CYNIC Actuators
**Implementation**: `scripts/hermes-x/core/`
**Status**: NATIVE SOVEREIGN NUCLEUS

- **Role**: The "Axiomatic Filter". If Hermes is the muscle, CYNIC is the **will** and the **conscience**.
- **Law**: These scripts define the *ethics* and *economics* of action.
- **Components**:
    - **Prompt Injection**: Overlays CYNIC's 6 Axioms onto the framework's `SOUL.md` at runtime.
    - **SOMA Gate**: Prevents the OS from saturating the sovereign hardware (Hardware-as-Law).
    - **Metabolic Ledger**: Measures the "Burn" (Axiome 5) by tracking token consumption and energy cost per verdict.

## 3. Separation of Concerns (SoC)

| Domain | Hermes (Agentic OS) | CYNIC (Sovereign Actuator) |
| :--- | :--- | :--- |
| **Identity** | Assistant / Explorer | Judge / Oracle / Sovereign |
| **Logic** | Tool-Use / Execution | Axiomatic Filtering / Gating |
| **Planning** | Internal Kanban | Task Dispatching (Kernel) |
| **Infrastructure** | Sandbox / Browser | Hardware (Native) / Energy |

## 4. Ecosystem Vision

The Hermes framework is the reference OS for Talaria. Its role is to provide the highest level of autonomous execution. CYNIC's role is to ensure this autonomy never deviates from the **Proof-of-History** and the **Axiomatic Constitution**. Other agentic frameworks may be studied and integrated in the future, but the "Code is Law" principle remains: the Engine executes, the Kernel judges, the Sovereign directs.
